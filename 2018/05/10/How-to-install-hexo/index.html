Step 1. Make your Github account.

Github is web-based hosting service for version control using git. Then you will have a question about git. Read the line below.

Git (/ɡɪt/[7]) is a version control system for tracking changes in computer files and coordinating work on those files among multiple people.

-Wikipedia

You should go github and make an account. It’s really easy and free. Than you can get an your own ID. You should remember that. It’s really important.

Step 2. Make Github Repository

You can make reposity after you make your own ID. And please set your reposity name with username.github.io

It is really important part of process. If you don’t do like that, git make error and you can’t make blog.

After that, you need to clone your git to your local system. Follow the line below.

$ git clone https://github.com/username/username.github.io

Step 3. Install HEXO

Now, you will install hexo. In this step, you need NODE. If you have not install it, go Node.js. And you need git, too. Git

After that, follow the line below.

$ npm install hero-cli -g

$ hexo init blog

$ cd blog

$ npm install

Then you can find _config.yml file. You can set lots of things with this. Here is something essential.

URL INFORMATION
Url: https://username.github.io

GIT INFORMATION
deploy:
type: git
repo: https://github.com/username/username.github.io

Step 4. Distribute on Github

Now, you will make distribution on your github. Follow the session below.

Resource generating by Hexo setting
$ hexo generate

Deploy Hexo on git
$ hexo deploy

IF YOU HAD ANY PLOBLEM, TRY THIS
$ hexo clean
$ hexo deploy -g

Step 5. Theme apply

Maybe you want awesome blog. It is hard to design your own, but you can use good examples on your site. You will clone the example following licences and apply it on GitHub.

You can find good examples here : https://hexo.io/themes/

Clone the theme repostory on your local, and setting _config.yml

theme: dxx

$ hexo clean
$ hexo deploy -g

Step 6. Writing

The purpose of making blog is maybe for writing your own ideas. For this, you should be used to Markdown Edit form. If you aren’t familiar with markdown, reference this

$ hexo new post [post_name]

It’s over. You can change the option with Markdown Editor. And just deploy it!

Please be happy with your blog-life.
